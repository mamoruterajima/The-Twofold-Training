# The-Twofold-Training
**〜 2つの意味でトレーニング 〜**

## コンセプト
極力AWSの無料枠内にて完結させること

## 📌 概要
本プロジェクトは、フィジーク（ボディビルの親戚）をたしなむ私が、トレーニングログを管理するためのWebアプリケーションです。
オンプレミスサーバーの保守運用、構築経験を持つが、モダンな **Infrastructure as Code** と **CI/CD パイプライン** を用いて AWS 上に構築することから、2つの意味でトレーニングとしました。
今後はボディメイクを補助する機能の追加と、他にも修めている馬術や武術の記録もできるような内容にしていこうと考えています。

## 🛠 使用技術
### バックエンド / フロントエンド
- **Ruby on Rails 8.0.1** (Propshaft / Importmaps)
- **SQLite3** (Active Storage / Action Text 動作確認済)

### Infrastructure (AWS)
- **Amazon EC2**: アプリケーション実行基盤
- **Amazon ECR**: Dockerイメージの管理
- **AWS Systems Manager (SSM)**: 安全なデプロイ操作（ポート22未開放）
- **Terraform**: インフラのコード化（ネットワーク、IAM、セキュリティグループ）

### CI/CD
- **GitHub Actions**: 
  - Dockerイメージのビルド & ECRへのPush
  - SSMによるEC2への自動デプロイ

## 🏗 インフラ構成のこだわり
- **Security First**: 
  SSH（22番ポート）をインターネットに公開せず、AWS SSM を経由してデプロイを実行する構成を採用。外部からの攻撃ベクトルを最小限に抑えています。
- **IaCによる再現性**: 
  Terraformを用いることで、環境の構築・破棄をコマンド一発で完結。コスト管理と環境の一貫性を両立しました。
- **Dockerコンテナによる可搬性**: 
  開発環境と本番環境の差異をなくすため、Rails 8 の Docker 構成をベースとしたコンテナ運用を行っています。
